docker — labo guidé

Labo : commandes Docker essentielles sous Ubuntu — exécution, interaction, cycle de vie, inspection et monitoring

Dans ce labo, vous allez d’abord vérifier votre environnement Ubuntu, lancer un conteneur Nginx, interagir avec lui et observer plusieurs commandes Docker utiles pour son cycle de vie, son inspection et son monitoring. Ensuite, vous préparerez une petite application Python de health check, vous la ferez fonctionner localement avec venv et pip, puis vous la conteneuriserez avec Docker. Enfin, vous récupérerez le fichier report.txt sur votre machine Ubuntu à l’aide d’un volume.

objectifs du labo

  • Utiliser les commandes Docker essentielles sur Ubuntu pour lancer, observer et arrêter un conteneur.
  • Interagir avec un conteneur à l’aide de docker exec et consulter ses journaux avec docker logs.
  • Comprendre plusieurs états d’un conteneur : pause, unpause, restart, stop et start.
  • Inspecter et surveiller un conteneur avec inspect, stats, diff et system df.
  • Construire une image Docker simple à partir d’un script Python et d’un Dockerfile.
  • Utiliser un volume Docker pour récupérer un fichier généré dans un conteneur sur la machine Ubuntu.

Partie 1 — Préparation et vérification des prérequis

Vérifier Python, venv et Docker

Avant de manipuler des conteneurs, vous devez confirmer que votre machine Ubuntu possède bien les outils nécessaires. Cette vérification évite de perdre du temps plus tard sur un simple problème d’installation ou de permission.

terminal — vérification des prérequis
etudiant@ubuntu:~$
1python3 --version
2python3 -m pip --version
3python3 -m venv --help
4docker --version
5docker ps

Partie 2 — Lancer un conteneur Nginx et interagir avec lui

Télécharger puis démarrer un service web simple

Cette première manipulation sert à observer un conteneur déjà prêt à l’emploi. Nginx est pratique pour débuter : l’image se télécharge vite, le démarrage est immédiat et le résultat est facile à vérifier.

terminal — lancer Nginx
etudiant@ubuntu:~$
1sudo docker pull nginx
2sudo docker run -d --name web1 -p 8080:80 nginx
3sudo docker ps

Partie 3 — Observer le cycle de vie, l’inspection et le monitoring

Tester les états d’un conteneur

Dans cette partie, vous allez manipuler plusieurs états possibles du même conteneur : pause, reprise, redémarrage, arrêt et démarrage. Le but est de comprendre qu’un conteneur déjà créé peut vivre plusieurs transitions sans être reconstruit.

terminal — cycle de vie du conteneur
etudiant@ubuntu:~$
1sudo docker pause web1
2sudo docker unpause web1
3sudo docker restart web1
4sudo docker stop web1
5sudo docker start web1
6sudo docker ps -a
Rappel : pause suspend le processus ; stop l’arrête ; start relance un conteneur déjà existant.

Inspecter le conteneur et surveiller quelques indicateurs

Un administrateur ne se contente pas de lancer un conteneur. Il doit aussi être capable de lire son état, de repérer les ports, de voir l’usage des ressources et d’identifier les changements internes réalisés depuis le démarrage.

terminal — inspection et monitoring
etudiant@ubuntu:~$
1sudo docker inspect -f '{{.Name}} | {{.State.Status}} | {{range $p, $c := .NetworkSettings.Ports}}{{$p}} {{end}}' web1
2sudo docker stats web1 --no-stream
3sudo docker diff web1
4sudo docker system df
Exemple de résultat attendu
/web1 | running | 80/tcp
CONTAINER ID   NAME   CPU %   MEM USAGE / LIMIT   MEM %
A /tmp/labo-docker.txt
TYPE            TOTAL     ACTIVE    SIZE      RECLAIMABLE
Capture obligatoire : montrez au minimum une sortie de docker inspect, une sortie de docker stats --no-stream et une sortie de docker diff web1.
ici la capture demandée.
ou collez une image enregistrée dans le presse-papiers avec Ctrl+V.

Partie 4 — Préparer et tester localement l’application Python

Créer un dossier de projet propre puis écrire le script

Le projet de cette seconde moitié du labo est un mini outil de health check. Il teste une URL, mesure le temps de réponse, affiche un verdict simple et écrit un rapport texte dans report.txt.

terminal — préparer le dossier de travail
etudiant@ubuntu:~$
1mkdir -p ~/docker-labo-healthcheck
2cd ~/docker-labo-healthcheck
3pwd
Contenu à enregistrer dans app.py
app.py
1import argparse
2import time
3from datetime import datetime
4
5import requests
6
7
8def check_url(url: str, timeout: int) -> dict:
9 start = time.perf_counter()
10 try:
11 response = requests.get(url, timeout=timeout)
12 elapsed_ms = (time.perf_counter() - start) * 1000
13 status = response.status_code
14 ok = 200 <= status < 400
15 verdict = "OK" if ok and elapsed_ms < 1000 else ("lent" if ok else "erreur")
16 return {
17 "url": url,
18 "status": status,
19 "elapsed_ms": round(elapsed_ms, 1),
20 "verdict": verdict,
21 "error": "",
22 }
23 except requests.exceptions.RequestException as exc:
24 elapsed_ms = (time.perf_counter() - start) * 1000
25 return {
26 "url": url,
27 "status": None,
28 "elapsed_ms": round(elapsed_ms, 1),
29 "verdict": "erreur",
30 "error": str(exc),
31 }
32
33
34def save_report(result: dict, filename: str) -> None:
35 now = datetime.now().strftime("%Y-%m-%d %H:%M:%S")
36 lines = [
37 f"date: {now}",
38 f"url: {result['url']}",
39 f"status: {result['status']}",
40 f"temps_ms: {result['elapsed_ms']}",
41 f"verdict: {result['verdict']}",
42 ]
43 if result["error"]:
44 lines.append(f"erreur: {result['error']}")
45
46 with open(filename, "w", encoding="utf-8") as file:
47 file.write("\n".join(lines) + "\n")
48

50def main() -> None:
51 parser = argparse.ArgumentParser(description="Mini health check d'une URL")
52 parser.add_argument("--url", default="https://example.com", help="URL à tester")
53 parser.add_argument("--timeout", type=int, default=5, help="Timeout en secondes")
54 parser.add_argument("--out", default="report.txt", help="Nom du rapport")
55 args = parser.parse_args()
56
57 result = check_url(args.url, args.timeout)
58
59 print("url:", result["url"])
60 print("status:", result["status"])
61 print("temps_ms:", result["elapsed_ms"])
62 print("verdict:", result["verdict"])
63 if result["error"]:
64 print("erreur:", result["error"])
65
66 save_report(result, args.out)
67 print("rapport:", args.out)
68

71 main()

Partie 5 — Construire l’image Docker de l’application

Écrire le Dockerfile correctement

Le Dockerfile décrit comment construire une image autonome. Ici, l’image doit contenir Python, la dépendance requests et votre script app.py.

Contenu à enregistrer dans Dockerfile
Dockerfile
1FROM python:3.12-slim
2WORKDIR /app
3COPY requirements.txt .
4RUN pip install --no-cache-dir -r requirements.txt
5COPY app.py .
6CMD ["python", "app.py"]
Attention : la ligne COPY requirements.txt . a bien une destination, et la commande de construction utilisera obligatoirement un point final pour indiquer le contexte courant.

Construire puis exécuter le conteneur

Une fois l’image construite, vous devez vérifier que le comportement en conteneur est cohérent avec le comportement observé localement après installation de la dépendance.

terminal — construire puis lancer l’image
(venv) etudiant@ubuntu:~/docker-labo-healthcheck$
1sudo docker build -t tp-python-requests .
2sudo docker run --rm tp-python-requests
3sudo docker run --rm tp-python-requests python app.py --url https://example.com --timeout 5
Exemple de résultat attendu

Partie 6 — Récupérer le rapport sur la machine Ubuntu

Utiliser un volume Docker pour sortir le fichier report.txt

Par défaut, un fichier créé dans le conteneur reste dans le conteneur. Avec un volume, vous pouvez faire apparaître le rapport directement sur la machine Ubuntu et le conserver même après la suppression du conteneur.

terminal — récupérer report.txt avec un volume
(venv) etudiant@ubuntu:~/docker-labo-healthcheck$
1mkdir -p reports
2sudo docker run --rm -v "$PWD/reports:/out" tp-python-requests python app.py --url https://example.com --out /out/report.txt
3ls -l reports
4cat reports/report.txt
Exemple de résultat attendu
report.txt
date: 2026-...
url: https://example.com
status: 200
temps_ms: ...
verdict: OK
